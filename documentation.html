Hasser CSS
A lite and flexible CSS framework. Meta text thanks to .has-text-meta, see text style. 


    <!-- hero can have classes 'has-hero', 'has-hero-top' or 'has-hero-bottom' -->

    <div class="has-hero">
        <div>
            <!-- the content of the hero here -->
        </div>
    </div>

This CSS-Only framework is mainly inspired by Skeleton CSS and born in the need to apply a custom style on the <input type="submit" /> or any button-like input of a Symfony form. So, without messing with the inner functionnality of any framework returning such a button not-accessible button-tag, it is possible to apply a button style it.

Then, much more classes came to be friends with the lonely background classes of buttons. And now we have a big but lite family of classes called Hasser CSS.

Plus: Tags have a default style to keep the harmony between them althrough the page.

Grid

The grid is wide of 6 columns, each column having class .col-*-6. The columns are dynamically resized based on the screen of the user.
To disable the the dynamic resizing of a column, append to its class -noresp, and the column will have the same size on all the screens.

Buttons

To set the main style of buttons

Base classes

Those classes are directly passed on tag that must have a button style


    <div>
        <button>no base class</button> <br/>
        <button class="is-button-default">.is-button-default</button>
        <button class="is-button-primary">.is-button-primary</button>
        <button class="is-button-success">.is-button-success</button>
        <button class="is-button-error">.is-button-error</button>
        <button class="is-button-warning">.is-button-warning</button>
    </div>


Wrapper classes

Those classes are passed to a wrapping tag and target the first child tag to apply the button style on


    <div>
        <div class="has-button-default"><button>.has-button-default</button></div>
        <div class="has-button-primary"><button>.has-button-primary</button></div>
        <div class="has-button-success"><button>.has-button-success</button></div>
        <div class="has-button-error"><button>.has-button-error</button></div>
        <div class="has-button-warning"><button>.has-button-warning</button></div>
    </div>

Utilities

There are more classes to shape the appearance of a tag

Padding
  • it is possible to set padding: .has-padding, .has-padding-large, .has-padding-larger
  • It is also possible to unset padding: .has-no-padding, .has-no-padding-left, .has-no-padding-right, .has-no-padding-bottom, .has-no-padding-top


Margin
  • it is possible to set margin: .has-margin, .has-margin-large, .has-margin-larger
  • It is also possible to unset margin: .has-no-margin, .has-no-margin-left, .has-no-margin-right, .has-no-margin-bottom, .has-no-margin-top


Miscellaneous
  • To activate animation in a block: .has-animation
  • To define a hero page (see the Hero cover): .has-hero, .has-hero-top, .has-hero-bottom
